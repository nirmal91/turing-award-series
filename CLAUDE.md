# Turing Award Series — Claude Memory

## What this project is
A series covering every Turing Award winner plus other interesting CS papers. One entry per week. Code, writeups, and one section per entry written by Nirmal (no AI).

## Repo
github.com/nirmal91/turing-award-series — branch: main

## Skills available locally
- `/new-chapter` — full workflow for adding a new chapter (research, code, README, LinkedIn, X thread, image spec, push)
- `/writing-fix-keep-same-tone` — fix grammar/spelling without changing voice or tone

Both stored at `/home/user/.claude/commands/`

## Chapter format (sections in this order)
1. My Take — written by Nirmal, never by AI, left as placeholder until he writes it
2. The Code — working implementation + how to run it
3. Full Worked Example — step-by-step walkthrough with real numbers, no skipped steps (added Week 03)
4. ELI5
5. ELI10
6. CS Graduate Level
7. Papers and Citations

## Code requirements for each chapter
- Self-contained Python (no external deps)
- Interactive REPL by default
- `--test` flag: 10+ test cases
- `--verbose` flag: shows internals
- Mirrors the historical concept, not just a simulation

## Nirmal's writing voice
- Short sentences, direct, no fluff
- No em dashes, no rule-of-three lists
- No AI-sounding phrases: "delve", "it's worth noting", "in today's world"
- Personal but not performative
- The rabbit holes / learning for its own sake framing is his

## LinkedIn post structure
1. Personal hook (missing learning, rabbit holes)
2. Series announcement + AI generated vs human written
3. 2-3 sentences on the winner in plain language
4. End: "Link in comments."

## X thread structure
- Tweet 1: Hook + series announcement + image
- Tweet 2: AI generated vs human written
- Tweet 3: Winner's contribution
- Tweet 4: Why it matters today / modern connection
- Tweet 5: github.com/nirmal91/turing-award-series

## Image spec (per chapter)
Split image: left = before the contribution, right = after. Dark background, monospace font. No headshots, no stock photos.
